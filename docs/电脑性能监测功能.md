# 电脑性能检测功能

App 展开 PC 设备卡片后，实时显示 CPU / 内存 / GPU / 磁盘 / 网络五维硬件监控数据，含 SVG 渐变折线图。

## 架构

```
App (Vue/Tauri)                     PC (Windows)
  ├─ fetchStats() 每 5s 调用 ──→  wol-agent :32249
  │                               GET /api/v1/stats
  │                               ← JSON (CPU/MEM/GPU/Disk/Net)
  ├─ buildStatsPath()              
  └─ SVG 折线图 + 渐变填充
```

数据采集在 PC 侧由 Go Agent 完成（gopsutil + wmic），App 侧负责差量计算、历史缓存、图表渲染。

```json
    "speed_mhz": 2667
  },
  "gpu": {
    "model": "NVIDIA GeForce RTX 3070",
    "vram_total_gb": 8,
    "usage_percent": 5
  },
  "disks": [
    {
      "name": "C:",
      "model": "KIOXIA-EXCERIA SSD",
      "total_gb": 931.1,
      "used_gb": 633.4,
      "read_bytes": 8232418304,
      "write_bytes": 1567452672,
      "partitions": [
        { "name": "C", "total_gb": 120, "used_gb": 104.4 },

    }
  ],
  "network": [
    {
      "name": "以太网",
      "desc": "Realtek PCIe 2.5GbE Family Controller",
      "recv_bytes": 205180428,
      "sent_bytes": 21388447001
    }
  ]
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `cpu.usage_percent` | 整体 CPU 使用率 %（`cpu.Percent(0, false)`） |
| `memory.ddr_type` | DDR 类型，来自 `wmic memorychip SMBIOSMemoryType`（26=DDR4, 34=DDR5, 24=DDR3） |
| `memory.speed_mhz` | 内存频率 MHz |
| `gpu.vram_total_gb` | 显存总量 GB（`wmic path Win32_VideoController AdapterRAM`） |
| `gpu.usage_percent` | GPU 使用率 %（`wmic path Win32_PerfFormattedData_GPUPerformanceCounters_GPUEngine`），可能为 null |
| `disks[].read_bytes` / `write_bytes` | 物理磁盘累计读写字节，App 做差量计算速率 |
| `disks[].partitions[].name` | 分区名，已去末尾 `\` 和 `:` |
| `network[].name` | 适配器名称（gopsutil，GBK 编码） |
| `network[].desc` | 硬件描述（`wmic path Win32_NetworkAdapter Description`），用于 UI 显示 |

## Agent 实现细节

### wmic 调用规范

- **必须加 `/format:csv`**：否则 wmic 输出 UTF-16 表格格式，`parseWmicCSV` 无法解析
- **`wmic path Win32_NetworkAdapter`**（非 `wmic nic`）：后者即使加 `/format:csv` 也输出 UTF-16
- **编码问题**：wmic 输出 GBK，gopsutil 返回 UTF-8 → 用 `InterfaceIndex`（数值）匹配，不用 `NetConnectionID`

### 关键函数

| 函数 | 来源 | 说明 |
|------|------|------|
| `collectCPU()` | gopsutil `cpu.Info` + `cpu.Percent` | 型号、核心数、使用率 |
| `collectMemory()` | gopsutil `mem.VirtualMemory` | 总量、已用、使用率 |
| `collectMemoryInfo()` | `wmic memorychip get SMBIOSMemoryType,Speed /format:csv` | DDR 类型 + 频率 |
| `collectGPU()` | `wmic path Win32_VideoController get Name,AdapterRAM /format:csv` | 两遍扫描：优先 VRAM>0 的真实 GPU，过滤虚拟适配器 |
| `collectGPUUsage()` | `wmic path Win32_PerfFormattedData_GPUPerformanceCounters_GPUEngine get UtilizationPercentage /format:csv` | 取全引擎最大 UtilizationPercentage |
| `collectDisks()` | gopsutil `disk.Partitions` + `disk.IOCounters` + WMI 关联类 | 分区→物理磁盘映射，IO 作用于物理磁盘（非逐分区重复） |
| `collectNetwork()` | gopsutil `net.IOCounters(true)` + `net.Interfaces` | 过滤 Loopback/Down/虚拟/WFP |
| `collectNicDescriptions()` | `wmic path Win32_NetworkAdapter get Description,InterfaceIndex /format:csv` | 硬件描述，按 InterfaceIndex 匹配 |
| `isVirtualNic()` | — | 过滤 Hyper-V/WSL/VMware/VirtualBox/Docker/TAP/TUN/Bluetooth 等 |

### 虚拟设备过滤

**GPU**：过滤名称含 `microsoft` / `virtual` / `oray` / `gameviewer` / `idd` 的适配器。两遍扫描：先找 `VRAMTotalGB > 0` 的真实 GPU，找不到则回退第一个非虚拟 GPU。

**网卡**：过滤名称或硬件描述含以下模式的适配器：
```
hyper-v, wsl, vmware, virtualbox, vbox, tap, tun, vethernet, docker,
bluetooth, isatap, teredo, 6to4, loopback, pseudo, miniport,
wan miniport, ras async, host-only, bridge, nat, virtual ethernet
```
同时过滤 WFP LightWeight Filter 等过滤器适配器（按 baseName 去重）。

### 磁盘 IO 计数器匹配

Windows 上 gopsutil `disk.IOCounters()` 返回 `\\.\PhysicalDriveN` 作为键。通过 WMI 关联类 `Win32_DiskDriveToDiskPartition` + `Win32_LogicalDiskToPartition` 将分区挂载点映射到物理磁盘序号，每个物理磁盘仅应用一次 IO 计数器（不按分区重复）。

## 前端实现

### 数据流

```
fetchStats(ip, port):
  1. httpGet(http://<ip>:<port>/api/v1/stats, socks5Proxy)
  2. 解析 JSON → statsInfo[ip]（模型名称等静态信息）
  3. 与上一次 statsRaw[ip] 比较 → 计算差量速率（B/s）
  4. push 到 statsHistory[ip]（StatsSnapshot[]，最多 60 点 = 5 分钟）
  5. 滑动窗口：shift 掉超出 60 点的旧数据
```

### 关键类型

```ts
interface StatsSnapshot {
  ts: number
  cpu: number          // CPU 使用率 %
  mem: number          // 内存使用率 %
  gpu: number | null   // GPU 使用率 %
  gpuMem: number | null
  disks: { name: string; readBps: number; writeBps: number }[]
  nets: { name: string; recvBps: number; sentBps: number }[]
}
```

差量计算：`(curBytes - prevBytes) / elapsedSeconds` → 字节/秒，存入 `readBps` / `writeBps` / `recvBps` / `sentBps`。

### 图表渲染

**SVG 结构**（参考组网流量图 `md-traffic-svg`）：
```svg
<svg viewBox="0 0 320 108" preserveAspectRatio="none" class="md-stats-svg">
  <defs>
    <linearGradient id="g-cpu-{ip}" x1="0" y1="0" x2="0" y2="1">
      <stop offset="0%" stop-color="#1976d2" stop-opacity="0.30"/>
      <stop offset="100%" stop-color="#1976d2" stop-opacity="0.02"/>
    </linearGradient>
  </defs>
  <!-- 填充区域（先渲染，在下层） -->
  <path d="..." fill="url(#g-cpu-{ip})"/>
  <!-- 折线（后渲染，在上层） -->
  <path d="..." fill="none" stroke="#1976d2" stroke-width="1.5"/>
  <line x1="0" y1="88" x2="320" y2="88" stroke="var(--md-border)" stroke-width="0.5"/>
</svg>
```

- `buildStatsPath(data, maxVal, 88, 320)` — 三次贝塞尔平滑曲线，与组网 `buildSmoothPath` 同算法
- `buildStatsAreaPath(...)` — 在曲线基础上闭合到 `chartH` 底线，用于渐变填充
- **viewBox**: `0 0 320 108`，**height**: 90px（同组网）。横轴线在 `y=88`（数据 0 点位置），chart-wrap 无左侧预留空间
- **横坐标**：左=旧数据 `-5m`（`left:2px` 贴边），右=最新 `now`（`right:2px` 贴边）
- **Y 轴标签**：仅最大值，绝对定位在 chart-wrap 左上角（`left:2px; top:1px`），不占用 SVG 内部空间；y=0 不显示标签，由横轴线隐式表达

### Y 轴自适应

| 图表 | Y 轴 max | 格式 |
|------|---------|------|
| CPU | `max(dataWindow)` | `%` |
| 内存 | `max(dataWindow)` GB | `GB`（1 位小数） |
| GPU | `max(dataWindow)` | `%` |
| 磁盘 | `max(readBps, writeBps)` | `B/s` → `KB/s` → `MB/s` → `GB/s`，同 `formatSpeedYStats` |
| 网络 | `max(recvBps, sentBps)` | `bps` → `Kbps` → `Mbps` → `Gbps`，`formatSpeedBps`（Byte→bit×8） |

**无最小值下限**：Y 轴最大值完全由当前窗口数据决定，数据滑出窗口后自动重新计算。不设 `Math.ceil(max/10)*10` 之类的倍数化。

### 横坐标标签

`statsXLabel(dataLen, idx)`：`idx=0`（最旧）→ `-5m`，`idx=dataLen-1`（最新）→ `now`。分钟标签用 `Math.round(min*2)/2` 四舍五入到 0.5 粒度（同组网 `formatXLabel`），消除 `-4.9m` 等小数。

### UI 布局（五板块）

每个板块：第一行 `md-stats-hdr`（左侧型号 + 右侧图例，flex-wrap 布局，标题过长时图例自动换至第二行居右），下一行 `md-stats-chart-wrap`（折线图，无横轴底线）。设备信息行（MAC/IP 等）移至展开区域底部，无分割线。

| 板块 | 标题（左侧） | 图例（右侧） |
|------|-------------|-------------|
| CPU | `Intel Core i7-10700F · 16 cores` | `● CPU 2%` |
| 内存 | `DDR4 16GB 2667MHz` | `● MEM 6.8G / 16G` |
| GPU | `NVIDIA GeForce RTX 3070` | `● GPU 5%` |
| 磁盘 | `KIOXIA-EXCERIA SSD` | `● R 0 B/s  ● W 0 B/s`（占用<80%蓝, 80-90%橙, ≥90%红） |
| 网卡 | `Realtek PCIe 2.5GbE Family Controller` | `● ↑ 0 bps  ● ↓ 0 bps` |

图例颜色：CPU `#1976d2`、MEM `#e57373`、GPU `#7c4dff`、Read/Sent ↑ `#43a047`、Write `#ff9800`、Recv ↓ `#1e88e5`。

### 磁盘排序

`diskList()` 对每个磁盘取其 ASCII 最小的分区名作为排序键（如 `C` < `D` < `E`），`.sort((a, b) => a.sortKey.localeCompare(b.sortKey))`，稳定不跳变。

## Agent 构建与部署

Agent 二进制位于 `D:\projects\projectsOpenwrt\luci-app-wolplus\wol-agent\`。

### 构建

```bash
cd D:/projects/projectsOpenwrt/luci-app-wolplus/wol-agent
go build -ldflags="-s -w" -o wol-agent.exe .
```

产物 `wol-agent.exe`（~6.4MB）。

### 部署（NSSM Windows 服务）

Agent 以 **NSSM Windows 服务** 形式运行，服务名 `WolAgent`，监听 `:32249`。

**首次安装**：
```batch
nssm install WolAgent "D:\tools\wolplus\wol-agent.exe" --port 32249
nssm set WolAgent AppDirectory "D:\tools\wolplus"
nssm set WolAgent Start SERVICE_AUTO_START
nssm set WolAgent Description "Wake-on-LAN Agent - remote status check and shutdown"
nssm start WolAgent
```

**已安装时替换更新**（需管理员权限）：
```powershell
# 1. 停服务
& "D:\tools\nssm-2.24\win64\nssm.exe" stop WolAgent

# 2. 确认已停（Get-Service WolAgent 状态为 Stopped）

# 3. 替换二进制
Copy-Item -Force "新wol-agent.exe" "D:\tools\wolplus\wol-agent.exe"

# 4. 启服务
& "D:\tools\nssm-2.24\win64\nssm.exe" start WolAgent

# 5. 验证
Invoke-RestMethod http://localhost:32249/api/v1/stats
```

**注意**：替换时目标文件被占用（`Copy-Item` 报 "being used by another process"），必须先停服务。NSSM 操作需管理员权限。

### 验证清单

部署后检查以下字段确认新版生效：
- `memory.ddr_type` 非空（如 `DDR4`）
- `memory.speed_mhz` 非 0（如 `2667`）
- `disks[].partitions[].name` 无末尾冒号（如 `C` 而非 `C:`）
- `network[].desc` 非空（如 `Realtek PCIe 2.5GbE Family Controller`）

## 已知限制

1. **GPU 显存占用不可获取**：wmic/WMI 无可靠的 VRAM 使用率计数器（`Win32_PerfFormattedData_GPUPerformanceCounters` 不稳定），仅显示显存总量和 GPU 使用率
2. **网卡名称 GBK 编码**：`network[].name` 在非 UTF-8 终端中可能乱码，UI 展示使用 `desc` 字段
3. **磁盘 IO 按物理磁盘汇总**：一个物理磁盘的多个分区共享同一组 IO 计数器
4. **Android 平台**：Agent 仅 Windows 可用
